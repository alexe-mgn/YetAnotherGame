Всё в папке Engine не предназначено для изменения конечным программистом.
Абсолютно всё, кроме содержимого этой папки, по идее пишется им.

Настройки движка и константы считываются из Engine/config.py,
которые, в свою очередь, сначала считываются из Engine/engine_config.py,
после чего перезаписываются значениями из game_config.py, его и стоит редактировать.
Эта система хлипкая и будет по-возможности изменена в близжайшем времени.

Практически все меню, игровые объекты и т.д создаются посредством создания в отдельном файле класса,
наследованного от базовых.

Все изображения обычно хранятся в классовых аттрибутах для экономии ОЗУ.
Загрузка осуществляется через функции Engine/loading.py,
многие классы имеют метод init_class, который выполняется сразу же после определения в том же файле.
Он предназначен для того,
чтобы загрузить графические модели и произвести необходимые для их корректного функционирования рассчёты.
